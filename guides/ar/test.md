## لماذا العزل مطلوب

عندما يعمل **Claude Code** على مهمة، فإنه يعمل بسرعة — يقرأ ويكتب ويعدل الملفات بنشاط. بدون عزل واضح، يمكن أن تنشأ عدة مشاكل:

- **التعارض مع عملك النشط.** إذا كان Claude يعمل على نفس الفرع الذي تعمل عليه، ستختلط التغييرات.
- **يحتار Claude بسبب التغييرات غير المكتملة.** الملفات غير المكتملة قد تسبب قرارات خاطئة.
- **لا توجد نقطة مراجعة واضحة.** بدون فرع منفصل، التراجع عن أخطاء Claude يصبح معقدًا.

## المفاهيم الأساسية

| المفهوم | الوصف |
|---------|-------|
| `git worktree add` | إضافة مساحة عمل جديدة |
| `git worktree list` | عرض جميع مساحات العمل النشطة |
| `git worktree remove` | إزالة مساحة عمل |

### مثال عملي

```bash
# إنشاء worktree مع فرع جديد
git worktree add ../my-project-issue-12 -b fix/issue-12

# الدخول إلى Worktree
cd ../my-project-issue-12

# تشغيل Claude Code
claude
```

> **ملاحظة:** المفاهيم في هذا الدليل عالمية — اختر الأداة التي تستخدمها وتخطَّ الإشارات إلى الأدوات التي لا تستخدمها.

## المتطلبات الأساسية

قبل البدء في هذا الدليل، يجب أن:

- تكون **مألوفًا بأساسيات Git** — commit, branch, push, pull, merge
- تكون **مرتاحًا في استخدام سطر الأوامر** — التنقل بين المجلدات، تحرير الملفات، تشغيل البرامج النصية
- يكون **Git مثبتًا** — `git --version` يجب أن يُرجع إصدارًا صالحًا
